# АгроБрянск — версия под GitHub Pages

Это полностью статическая версия сайта под GitHub Pages.

## Что уже работает на GitHub Pages
- красивый адаптивный интерфейс;
- мобильная версия;
- карточки компаний;
- сравнение предприятий;
- база знаний;
- аналитика;
- погода по районам Брянской области через Open-Meteo;
- чат-бот по данным сайта;
- формы добавления форума, товаров, кооперации, меток и сообщений оператору.

## Что нужно для общих данных между всеми пользователями
GitHub Pages сам по себе не запускает серверный код. Поэтому для выполнения требований:
- аккаунты;
- общий форум;
- общие метки;
- общий маркетплейс;
- общая кооперация;
- главный аккаунт администратора;
- удаление данных администратором;
- общая обратная связь;

используется **Supabase** как внешний backend.

## Быстрый запуск
1. Создайте репозиторий или загрузите файлы в существующий.
2. Откройте `assets/js/config.js`.
3. Вставьте свои `supabaseUrl` и `supabaseAnonKey`.
4. В Supabase откройте SQL Editor и выполните `assets/sql/supabase_schema.sql`.
5. Включите Email/Password в Authentication > Providers.
6. Включите Realtime для таблиц, перечисленных в SQL-файле.
7. В GitHub Pages публикуйте сайт из ветки `main` и корня репозитория.

## Главный аккаунт
Главный аккаунт администратора: `agrobransk@inbox.ru`

Чтобы он получил права администратора:
1. зарегистрируйте этот email через сайт или через Supabase Auth;
2. подтвердите email, если у вас включено подтверждение;
3. после входа сайт автоматически распознает его как главный аккаунт.

## Откуда берутся данные
- Погода: Open-Meteo API.
- Компании, база знаний, резервные цены: локальные JSON-данные внутри `assets/js/data.js`.
- Общие пользовательские данные: Supabase.

## Как обновлять компании и цены без переписывания кода
В `assets/js/config.js` есть поля:
- `pricesFeedUrl`
- `enterprisesFeedUrl`

Если указать ссылку на внешний JSON, сайт будет брать актуальные данные оттуда.
Это удобно, если вы захотите обновлять цены и каталоги через Google Sheets, Supabase table view, отдельный JSON-файл или GitHub raw.

## Структура проекта
- `index.html` — основная страница
- `assets/css/style.css` — стили
- `assets/js/config.js` — конфиг
- `assets/js/config.example.js` — пример конфига
- `assets/js/data.js` — резервные данные
- `assets/js/app.js` — логика сайта
- `assets/sql/supabase_schema.sql` — структура базы и политики доступа

## Важно
Если не заполнить `config.js`, сайт всё равно откроется, но будет в демо-режиме без общей синхронизации между пользователями.


## Пошагово: как подключить Supabase
1. Зайдите в Supabase и создайте новый проект.
2. Дождитесь, пока проект станет активным.
3. Откройте **Project Settings** → **Data API** или **API** и скопируйте:
   - **Project URL**
   - **anon / publishable key**
4. Откройте файл `assets/js/config.js`.
5. Вставьте значения в поля `supabaseUrl` и `supabaseAnonKey`.

Пример:
```js
window.APP_CONFIG = {
  supabaseUrl: 'https://ВАШ-ПРОЕКТ.supabase.co',
  supabaseAnonKey: 'ВАШ_ПУБЛИЧНЫЙ_КЛЮЧ',
  pricesFeedUrl: '',
  enterprisesFeedUrl: '',
  defaultMapCenter: [53.242377, 34.366828],
  adminEmail: 'agrobransk@inbox.ru'
};
```

### Важно
В браузерный файл `config.js` вставляется только **anon/publishable key**.
**service_role key** туда вставлять нельзя.

## Как сделать общие данные для всех пользователей
1. В Supabase откройте **SQL Editor**.
2. Скопируйте содержимое файла `assets/sql/supabase_schema.sql`.
3. Выполните SQL целиком.
4. В **Authentication** включите вход по **Email/Password**.
5. Зарегистрируйте через сайт администратора `agrobransk@inbox.ru`.
6. Войдите под этим email — сайт автоматически даст ему права главного аккаунта.
7. Убедитесь, что Realtime включён для таблиц из SQL-схемы.

После этого форум, товары, кооперация, метки, сообщения операторам и очки будут храниться в одной общей базе и станут видны всем пользователям.

## Как включить GitHub Pages
1. Загрузите файлы проекта в репозиторий GitHub.
2. Откройте репозиторий → **Settings** → **Pages**.
3. В блоке **Build and deployment** выберите **Deploy from a branch**.
4. В поле **Branch** выберите ветку `main` и папку `/root`.
5. Сохраните настройки.
6. Подождите публикации и откройте адрес сайта, который покажет GitHub Pages.

## Как обновить сайт после изменений
1. Меняете файлы в репозитории.
2. Делаете commit и push.
3. GitHub Pages автоматически публикует обновлённую статическую версию сайта.
